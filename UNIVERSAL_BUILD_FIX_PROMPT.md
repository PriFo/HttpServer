# Универсальный промпт для сборки и генерации плана исправления ошибок

Скопируй этот промпт в новый чат для автоматической сборки проекта и создания плана исправления ошибок:

---

Выполняй команды в терминале (PowerShell), когда это необходимо.

Забилди бекенд проект Go и запомни ошибки. Создай пошаговый план исправления ошибок для всех ошибок при сборке. Сделай промт для каждого шага. Сделай желательно так, если это реально, чтобы я мог каждый шаг запустить параллельно и сделай каждый шаг - исправление одного Go файла и не делай исправление файла построчно, когда условно первый шаг исправляет 30 строку файла, а 4 шаг исправляет 120 - это не правильно. Укажи в промте, какую ошибку выдает компилятор, расскажи все тонкости этой ошибки, но не пиши, как её исправить, а попроси исправить в промте к шагу, потом пусть уже в самом этой задачи, в самом промте думает над решением. Также добавь в промт объяснение, что некоторые зависимые файлы а-ля handler-repository писались отдельно, т.е. handler зависит от repository, но repository не существовал(Но он уже может быть создан), поэтому могут быть простые ошибки, к примеру, в наименование файла, но может быть действительно просто не написан какой-то файл и его надо написать, но точно убедись в этом. В шагах в плане должны быть только промты.

Сгенерируй план с помощью Python скрипта generate_build_fix_plan.py, не надо выдумывать и писать, как исправить ошибку - просто с помощью скрипта(Переиспользуй скрипт) сделай промт с ссылкой на файл, ошибками компилятора и примечанием ниже. Также в конце сообщи, сколько ошибок и промтов.

Добавь в конец промтов:

- Некоторые зависимые файлы, могли создаваться отдельно, т.е. Handler может зависеть от repository, но repository мог не существовать на момент написания handler

- Могут быть простые ошибки в наименовании файлов при импорте

- Но может быть действительно отсутствующий файл - убедись на 150%, что файл действительно отсутствует, прежде чем создавать его

- Проверь все импорты и убедись, что они указывают на существующие файлы с правильными именами

- Проверь после исправление компиляцию файла напрямую и, если есть ошибки - исправь их. Делай циклы "Проверка компиляции - исправление ошибки" пока не будет ошибок. Если после компиляции файла остались ошибки - сообщи об этом.

Если бекенд забилдился без ошибок - ничего не делай, останови и сообщи это в чат.

---

## Инструкция по использованию:

1. Скопируй промпт выше (текст между "---")
2. Вставь в новый чат
3. AI автоматически:
   - Соберет проект командой `go build ./...`
   - Сохранит ошибки в `build_errors_full.log`
   - Запустит скрипт `generate_build_fix_plan.py` для генерации плана
   - Создаст файл `build_fix_plan.md` с планом
   - Создаст отдельные промпты в `build_fix_prompts/`
   - Сообщит статистику ошибок

## Что будет создано:

- `build_errors_full.log` - все ошибки компиляции
- `build_fix_plan.md` - план с промптами для каждого файла
- `build_fix_prompts/step_XX_*.txt` - отдельные промпты для каждого шага

## Пример использования:

Просто скопируй текст промпта (между разделителями "---") и вставь в новый чат. AI выполнит все автоматически.
