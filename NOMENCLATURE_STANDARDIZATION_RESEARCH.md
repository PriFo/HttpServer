# Исследование возможностей DaData и Adata для нормализации номенклатуры

## Дата исследования
2025-01-XX

## Цель исследования
Определить, могут ли сервисы DaData.ru и Adata.kz использоваться для нормализации номенклатуры товаров и услуг.

## Результаты исследования

### DaData.ru

**Доступные API:**
- `/suggest/party` - подсказки по организациям (компаниям)
- `/suggest/address` - подсказки по адресам
- `/suggest/fio` - подсказки по ФИО
- `/suggest/bank` - подсказки по банкам
- `/clean/address` - стандартизация адресов
- `/clean/phone` - стандартизация телефонов
- `/clean/name` - стандартизация ФИО

**Выводы:**
- DaData.ru **НЕ предоставляет** API для нормализации номенклатуры товаров
- Сервис специализируется на стандартизации адресов, телефонов, ФИО и данных организаций
- В документации DaData отсутствует упоминание эндпоинта `/suggest/product` или аналогичного
- DaData фокусируется на данных о контрагентах, адресах и контактах, но не на товарах

**Рекомендации:**
- Для нормализации номенклатуры товаров необходимо использовать другие решения:
  - Специализированные сервисы нормализации товаров
  - Генеративные AI модели (уже интегрированы в систему)
  - Собственные алгоритмы на основе классификаторов (ОКПД2, ТН ВЭД и т.д.)

### Adata.kz

**Доступные API:**
- `/api/company/info` - информация о компании по БИН/ИИН
- API для получения налоговой информации
- API для получения факторов риска
- API для проверки благонадежности

**Выводы:**
- Adata.kz **НЕ предоставляет** API для нормализации номенклатуры товаров
- Сервис специализируется исключительно на информации о компаниях (контрагентах)
- В документации Adata.kz отсутствует упоминание API для работы с товарами или номенклатурой
- Adata.kz фокусируется на данных о контрагентах, но не на товарах

**Рекомендации:**
- Для нормализации номенклатуры товаров в Казахстане необходимо использовать другие решения:
  - Специализированные сервисы нормализации товаров
  - Генеративные AI модели (уже интегрированы в систему)
  - Собственные алгоритмы на основе казахстанских классификаторов

## Общие выводы

1. **DaData.ru и Adata.kz НЕ подходят для нормализации номенклатуры товаров**
   - Оба сервиса специализируются на данных о контрагентах
   - Нет API для работы с товарами или номенклатурой

2. **Текущая система уже имеет решение для нормализации номенклатуры:**
   - Генеративные AI провайдеры (OpenRouter, Hugging Face, Arliai, Eden AI)
   - Система нормализации номенклатуры через `MultiProviderAINormalizer`

3. **Рекомендации по дальнейшему развитию:**
   - Продолжать использовать генеративные AI для нормализации номенклатуры
   - Рассмотреть интеграцию специализированных сервисов нормализации товаров (если такие найдутся)
   - Использовать классификаторы (ОКПД2, ТН ВЭД) для категоризации товаров
   - Развивать собственные алгоритмы нормализации на основе накопленных данных

## Заключение

Интеграция DaData и Adata для нормализации контрагентов является правильным решением и успешно реализована. Однако для нормализации номенклатуры товаров эти сервисы не подходят. Текущая система с генеративными AI провайдерами остается оптимальным решением для нормализации номенклатуры.
