# Инструкция по запуску серверов

## Проблема
Фронтенд работает на порту 3000, но не может подключиться к бэкенду на порту 9999.

## Решение: Запуск обоих серверов

### Способ 1: Использовать готовый скрипт (РЕКОМЕНДУЕТСЯ)

**Windows (PowerShell):**
```powershell
.\start-all.ps1
```

**Windows (CMD):**
```cmd
start-all.bat
```

Этот скрипт автоматически запустит:
- ✅ Бэкенд на порту 9999
- ✅ Фронтенд на порту 3000

### Способ 2: Запуск вручную

#### Шаг 1: Запустить бэкенд

Откройте **первый терминал** и выполните:

```powershell
cd E:\HttpServer
$env:ARLIAI_API_KEY='597dbe7e-16ca-4803-ab17-5fa084909f37'
go run -tags no_gui main_no_gui.go
```

Или используйте готовый exe:
```powershell
.\start-backend-exe.bat
```

**Ожидаемый вывод:**
```
Сервер запущен на порту 9999
API доступно по адресу: http://localhost:9999
```

#### Шаг 2: Запустить фронтенд

Откройте **второй терминал** и выполните:

```powershell
cd E:\HttpServer\frontend
npm run dev
```

**Ожидаемый вывод:**
```
✓ Ready in XXXms
- Local:        http://localhost:3000
```

### Способ 3: Проверка статуса

После запуска проверьте:

1. **Бэкенд работает?**
   ```powershell
   Invoke-WebRequest -Uri http://localhost:9999/api/dashboard/stats -UseBasicParsing
   ```
   Должен вернуть статус 200.

2. **Фронтенд работает?**
   Откройте в браузере: http://localhost:3000

3. **Порты заняты?**
   ```cmd
   netstat -ano | findstr :9999
   netstat -ano | findstr :3000
   ```

## Устранение проблем

### Порт 9999 уже занят
```cmd
netstat -ano | findstr :9999
taskkill /F /PID <номер_процесса>
```

### Порт 3000 уже занят
```cmd
netstat -ano | findstr :3000
taskkill /F /PID <номер_процесса>
```

### Бэкенд не отвечает
1. Проверьте логи в окне терминала бэкенда
2. Убедитесь, что базы данных существуют (1c_data.db, normalized_data.db, service.db)
3. Проверьте, что Go установлен и доступен: `go version`

## После успешного запуска

1. Откройте браузер: http://localhost:3000
2. Страница должна загрузиться без ошибок
3. Данные должны подгружаться с бэкенда
