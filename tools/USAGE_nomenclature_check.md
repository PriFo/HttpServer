# Использование утилиты проверки номенклатур

## Быстрый старт

1. Скомпилируйте утилиту:
```bash
go build -o check_nomenclature_count.exe tools/check_nomenclature_count.go
```

2. Запустите проверку:
```bash
# Для клиента
.\tools\check_nomenclature_count.exe -db "путь\к\базе.db" -client 1

# Для проекта
.\tools\check_nomenclature_count.exe -db "путь\к\базе.db" -client 1 -project 1

# С деталями
.\tools\check_nomenclature_count.exe -db "путь\к\базе.db" -client 1 -details
```

## Что показывает утилита

Утилита показывает:
- Количество нормализованных номенклатур (из таблицы `normalized_data`)
- Количество исходных номенклатур (из таблиц `catalog_items` и `nomenclature_items`)
- Общее количество номенклатур

## Важно для проверки исправления

После исправления багфикса, базы данных, которые содержат **и** таблицу `normalized_data`, **и** исходные таблицы, должны показывать **все** номенклатуры из обоих источников.

**До исправления:** показывались только нормализованные номенклатуры (1 штука)
**После исправления:** должны показываться все номенклатуры (нормализованные + исходные)
