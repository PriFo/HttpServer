Исправь ошибку циклического импорта в пакете `internal/api/handlers/classification`.

## Файл для исправления:
`internal/api/handlers/classification/handler.go`

## Ошибки компилятора:

Ошибка 1:
  Файл: internal/api/handlers/classification/handler.go
  Строка: 9, Колонка: 2
  Сообщение компилятора: found packages handlers (base.go) and server (classification_legacy.go) in E:\HttpServer\server\handlers
  Пакет: httpserver

Ошибка 2 (циклический импорт):
  Цепочка импортов:
  - httpserver/server (из main.go)
  - httpserver/internal/api/routes (из server.go)
  - httpserver/internal/api/handlers/classification (из classification_routes.go)
  - httpserver/server/handlers (из handler.go)
  - httpserver/internal/container (из upload_v2_legacy.go)
  - httpserver/internal/api/handlers/classification (из classification_init.go)
  
  Сообщение: import cycle not allowed

## Объяснение типов ошибок (тонкости):


Ошибка "found packages X and Y" означает, что в одной директории находятся файлы с разными именами пакетов. В Go все файлы в одной директории должны принадлежать одному пакету.

Тонкости этой ошибки:
- В директории `server/handlers` есть файлы с разными package декларациями
- Файл `base.go` объявляет `package handlers`
- Файл `classification_legacy.go` объявляет `package server`
- Это нарушает правило Go о том, что все файлы в директории должны иметь одинаковое имя пакета
- Может быть результатом рефакторинга, когда файлы были перемещены, но package декларации не обновлены


Ошибка "import cycle not allowed" означает, что обнаружена циклическая зависимость между пакетами.

Тонкости этой ошибки:
- Go не позволяет циклических импортов между пакетами
- Пакет A импортирует B, B импортирует C, C импортирует A - это цикл
- Циклические зависимости указывают на проблемы в архитектуре
- Решения: вынести общий код в отдельный пакет, использовать интерфейсы, разделить пакеты на более мелкие модули
- Может потребоваться рефакторинг структуры пакетов

## Важные примечания:

- Некоторые зависимые файлы могли создаваться отдельно, т.е. Handler может зависеть от repository, но repository мог не существовать на момент написания handler

- Могут быть простые ошибки в наименовании файлов при импорте

- Но может быть действительно отсутствующий файл - убедись на 150%, что файл действительно отсутствует, прежде чем создавать его

- Проверь все импорты и убедись, что они указывают на существующие файлы с правильными именами

- Проверь после исправления компиляцию файла напрямую и, если есть ошибки - исправь их. Делай циклы "Проверка компиляции - исправление ошибки" пока не будет ошибок. Если после компиляции файла остались ошибки - сообщи об этом.
