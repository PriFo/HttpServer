## Объяснение типов ошибок (тонкости):


Ошибка "redeclared in this block" означает, что символ (тип, функция, переменная) объявлен дважды в одной области видимости.

Тонкости этой ошибки:
- В Go нельзя объявлять один и тот же символ дважды в одном пакете
- Может быть дублирование объявления в разных файлах одного пакета
- Может быть дублирование в одном файле
- Нужно удалить одно из объявлений или переименовать одно из них
- Может быть случайное копирование кода, которое привело к дублированию
- Может быть рефакторинг, когда код был перемещен, но старое объявление не удалено


Ошибка "unknown field" означает, что в структуре пытаются использовать поле, которое не определено в типе структуры.

Тонкости этой ошибки:
- Поле может быть определено в другой версии структуры
- Поле может быть переименовано, но код не обновлен
- Поле может быть удалено при рефакторинге
- Может быть опечатка в имени поля
- Поле может быть определено в другом типе структуры
- Нужно проверить определение структуры и убедиться, что поле существует


Ошибка "already declared" означает, что метод уже объявлен для этого типа.

Тонкости этой ошибки:
- В Go нельзя объявлять один и тот же метод дважды для одного типа
- Может быть дублирование метода в разных файлах одного пакета
- Может быть дублирование в одном файле
- Нужно удалить одно из объявлений метода
- Может быть случайное копирование кода


Ошибка "declared and not used" означает, что переменная объявлена, но не используется в коде.

Тонкости этой ошибки:
- В Go неиспользуемые переменные считаются ошибкой компиляции
- Переменная может быть оставлена после рефакторинга
- Переменная может быть объявлена, но код, использующий её, был удален
- Можно удалить переменную или использовать её (например, присвоить _)
- Может быть опечатка - переменная должна использоваться, но не используется


Ошибка "undefined" означает, что компилятор не может найти определение символа (переменной, функции, типа, пакета).

Тонкости этой ошибки:
- Символ может быть не объявлен в текущем пакете или в импортированных пакетах
- Неправильный импорт пакета - возможно, пакет импортирован, но символ в нем не существует или имеет другое имя
- Символ может быть объявлен в другом файле того же пакета, но не экспортирован (начинается с маленькой буквы) - в Go экспортируются только символы с заглавной буквы
- Опечатка в имени символа - очень частая причина
- Файл с определением может не быть включен в компиляцию (не в той директории или не имеет правильного package)
- Тип может быть определен в другом пакете, но не импортирован
- Может быть проблема с циклическими зависимостями между пакетами
- Поле может быть удалено из структуры, но код все еще пытается его использовать

## Важные примечания:

- Некоторые зависимые файлы могли создаваться отдельно, т.е. Handler может зависеть от repository, но repository мог не существовать на момент написания handler

- Могут быть простые ошибки в наименовании файлов при импорте

- Но может быть действительно отсутствующий файл - убедись на 150%, что файл действительно отсутствует, прежде чем создавать его

- Проверь все импорты и убедись, что они указывают на существующие файлы с правильными именами

- Проверь после исправления компиляцию файла напрямую и, если есть ошибки - исправь их. Делай циклы "Проверка компиляции - исправление ошибки" пока не будет ошибок. Если после компиляции файла остались ошибки - сообщи об этом.
