# Выбор проекта на странице процессов

## Описание

Добавлена возможность выбора проекта на странице процессов для запуска нормализации по всем базам данных проекта сразу.

## Изменения

### 1. Создан компонент `ProjectSelector`
- **Файл:** `frontend/components/project-selector.tsx`
- **Функциональность:**
  - Выбор клиента из списка
  - Выбор проекта выбранного клиента
  - Автоматическая загрузка проектов при выборе клиента
  - Кнопка обновления списка
  - Формат значения: `"clientId:projectId"`

### 2. Обновлена страница процессов
- **Файл:** `frontend/app/processes/page.tsx`
- **Изменения:**
  - Добавлен компонент `ProjectSelector` рядом с `DatabaseSelector`
  - Добавлен URL параметр `project` (формат: `"clientId:projectId"`)
  - При выборе проекта очищается выбор базы данных
  - При выборе базы данных очищается выбор проекта
  - Оба селектора работают независимо

### 3. Обновлен компонент `NormalizationProcessTab`
- **Файл:** `frontend/components/processes/normalization-process-tab.tsx`
- **Изменения:**
  - Добавлен пропс `project` (опциональный)
  - Парсинг проекта из пропса в `clientId` и `projectId`
  - Обновлена логика `handleStart`:
    - Если выбран проект, запускается нормализация для всех активных БД проекта (`all_active: true`)
    - Если выбрана база данных, работает как раньше
  - Обновлено описание: показывает информацию о проекте или базе данных
  - Обновлен текст кнопки: "Запустить нормализацию для всех БД проекта" при выборе проекта
  - Кнопка запуска активна, если выбран проект или база данных

## Использование

### Запуск нормализации для всех БД проекта:

1. Откройте страницу `/processes`
2. Выберите клиента в первом селекторе
3. Выберите проект во втором селекторе
4. Нажмите "Запустить нормализацию для всех БД проекта"

### Запуск нормализации для конкретной БД:

1. Откройте страницу `/processes`
2. Выберите базу данных в селекторе "Выберите базу данных"
3. Нажмите "Запустить нормализацию"

## API

При выборе проекта используется endpoint:
```
POST /api/clients/{clientId}/projects/{projectId}/normalization/start
```

С телом запроса:
```json
{
  "all_active": true,
  "use_kpved": false,
  "use_okpd2": false
}
```

## Примечания

- Выбор проекта и выбор базы данных взаимоисключающие
- При выборе проекта обрабатываются все активные базы данных проекта
- Статус нормализации проверяется через endpoint проекта, если проект выбран
- Обратная совместимость сохранена: можно использовать старый способ с выбором базы данных
